# 云端存储方案 - 飞书多维表格

## 方案概述

使用飞书多维表格作为云端数据库，实现：
- ✅ 多设备数据同步
- ✅ 每日持仓自动备份
- ✅ 历史数据云端存储
- ✅ 交易日志同步

## 对接方式

### 方案A：飞书开放平台API（推荐）

1. **申请应用**
   - 访问 https://open.feishu.cn/
   - 创建企业自建应用
   - 获取 App ID 和 App Secret

2. **配置权限**
   - `bitable:record:read`
   - `bitable:record:write`

3. **获取多维表格Token**
   - 创建多维表格
   - 复制表格链接中的 token
   - 格式：`https://open.feishu.cn/base/XXX`

4. **API调用示例**
   ```javascript
   // 读取数据
   fetch('https://open.feishu.cn/open-apis/bitable/v1/apps/{app_token}/tables/{table_id}/records', {
       headers: {
           'Authorization': 'Bearer ' + access_token
       }
   });
   
   // 写入数据
   fetch('https://open.feishu.cn/open-apis/bitable/v1/apps/{app_token}/tables/{table_id}/records', {
       method: 'POST',
       headers: {
           'Authorization': 'Bearer ' + access_token,
           'Content-Type': 'application/json'
       },
       body: JSON.stringify({
           fields: {
               "日期": "2026-03-11",
               "总资产": 395000
           }
       })
   });
   ```

### 方案B：手动导入导出（过渡方案）

1. 每日收盘后，导出 CSV 数据
2. 上传到飞书多维表格
3. 我提供数据转换脚本

## 同步流程

### 每日自动同步（计划）

```
20:30 收盘后
    ↓
获取当日净值/股价
    ↓
计算总资产、盈亏
    ↓
写入飞书多维表格
    ↓
发送飞书通知给用户
    ↓
备份到 GitHub（双保险）
```

### 多设备访问

**设备A（手机）**
```
打开网页 → 读取飞书API → 显示最新数据
```

**设备B（电脑）**
```
打开网页 → 读取飞书API → 显示相同数据
```

**数据一致性**
- 飞书作为单一数据源
- 所有设备从同一源头读取

## 备选方案：Supabase

如果飞书方案有问题，备选方案：

### Supabase 配置

1. 注册 https://supabase.com/
2. 创建项目（免费）
3. 创建数据表（同上结构）
4. 获取 API URL 和 Key
5. 网页直接对接

### 优势
- 国际服务，文档完善
- PostgreSQL 数据库
- 实时订阅（一个设备改，其他设备秒同步）

### 劣势
- 国内访问可能较慢
- 需要科学上网配置
