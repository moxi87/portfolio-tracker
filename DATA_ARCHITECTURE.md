# Portfolio Pro 数据架构设计 v1.0

## 目标
- 持仓数据真实可靠，非随机生成
- 多数据源交叉验证
- 旧数据写死，新数据实时获取
- 支持比亚迪(002594)、中科曙光(603019)、洛阳钼业(603993)

## 数据源选择（按可靠性排序）

### 1. 新浪财经API ⭐ 首选
- URL: `https://hq.sinajs.cn/list=sh{code}` 或 `sz{code}`
- 优点: 20年历史，极其稳定，毫秒级延迟
- 数据: 实时价格、涨跌幅、成交量、买卖五档
- 限制: 需要处理JSONP格式，IP限流（每秒10次足够）

### 2. 腾讯股票API ⭐ 备用
- URL: `https://qt.gtimg.cn/q=sh{code},sz{code}`
- 优点: 数据全面，包含PE/PB等财务指标
- 数据: 实时价格、涨跌幅、市值、财务指标
- 限制: 格式较乱，需要正则解析

### 3. 东方财富API ⭐ 历史数据
- 通过akshare库调用
- 优点: 历史K线、分钟级数据、财务数据
- 数据: 历史行情、技术指标、财务报表
- 限制: 需要Python后端

### 4. 跨市场验证（数据交叉对比）
- 同时请求新浪+腾讯
- 对比价格差异（>0.5%报警）
- 取平均值作为最终数据

## 技术实现方案

### 方案A: 纯前端（快速部署）
- 直接从前端调用新浪/腾讯API（CORS需处理）
- 使用JSONP绕过跨域
- 数据缓存在localStorage
- 缺点: 数据源单一，无法交叉验证

### 方案B: Python后端（推荐）⭐
- 部署在用户的OpenClaw环境
- 定时任务抓取数据
- 交叉验证逻辑
- 提供API给前端
- 优点: 可靠、可控、可扩展

### 方案C: Serverless函数
- Vercel/Cloudflare Workers
- 免费额度足够
- 全球CDN加速
- 缺点: 需要额外账号

## 推荐实施方案

采用**方案B: Python后端 + 定时任务**

### 组件清单
1. `data_fetcher.py` - 数据抓取服务
2. `data_validator.py` - 交叉验证逻辑
3. `cache_manager.py` - 缓存管理
4. `api_server.py` - Flask/FastAPI提供接口
5. `cron_sync.py` - 定时同步任务

### 部署步骤
1. 安装依赖: `pip install akshare requests flask`
2. 配置数据源优先级
3. 启动API服务
4. 配置定时任务（每5秒更新实时数据）
5. 前端对接新API

## 风险控制

### 数据异常处理
- 价格变动>10%: 标记待确认
- 数据源差异>1%: 报警并取平均值
- 数据源全部失败: 使用缓存数据，标记"数据延迟"

### 限流保护
- 新浪: 每秒10次
- 腾讯: 每秒5次
- 东财: 每分钟100次

### 降级策略
1. 主数据源失败 → 切备用源
2. 全部实时源失败 → 使用缓存（15分钟内有效）
3. 缓存过期 → 显示最后已知价格，标记"数据暂停"
