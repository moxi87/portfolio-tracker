# 持仓跟踪系统 - 部署说明

## 📁 已生成文件

```
portfolio-tracker/
├── index.html              # 主页面（完整的持仓追踪网页）
├── README.md               # 项目说明文档
├── portfolio-data.json     # 持仓数据文件
└── .github/
    └── workflows/
        └── update-data.yml # 自动更新工作流
```

## 🚀 部署步骤（3分钟完成）

### 1. 注册 GitHub 账号
访问 https://github.com/signup 注册

### 2. 创建仓库
- 点击右上角 **+** → **New repository**
- Repository name: `portfolio-tracker`
- 选择 **Public**
- 勾选 **Add a README file**
- 点击 **Create repository**

### 3. 上传文件
在仓库页面：
1. 点击 **Add file** → **Upload files**
2. 选择生成的 4 个文件（index.html、README.md、portfolio-data.json、update-data.yml）
3. Commit changes

**文件夹结构注意**：
- `.github/workflows/update-data.yml` 需要放在对应文件夹中
- 可以在本地先创建好文件夹再上传，或使用 GitHub 网页逐层创建

### 4. 开启 GitHub Pages
1. 进入仓库 **Settings** → **Pages**
2. **Source** 选择 **Deploy from a branch**
3. **Branch** 选择 **main** / **(root)**
4. 点击 **Save**
5. 等待 1-2 分钟后刷新，会显示访问链接如：
   `https://你的用户名.github.io/portfolio-tracker/`

## 📱 访问方式

部署成功后，你可以：
1. **电脑浏览器**直接访问 GitHub Pages 链接
2. **手机浏览器**收藏该链接到桌面
3. **飞书**中发送链接，点击即可打开

## 🔄 每日更新方案

我为你设计了三种更新方式：

### 方案 A：我帮你更新（推荐）
每天晚上收盘后（约 20:30），我会：
1. 抓取你持仓标的当日收盘价
2. 计算最新市值和盈亏
3. 更新 portfolio-data.json
4. 推送到你的 GitHub 仓库
5. 网页自动显示最新数据

**你需要做的**：
- 提供 GitHub 仓库的写入权限（加我协作者），或
- 每天晚上在飞书问我："更新持仓数据"

### 方案 B：GitHub Actions 自动更新
已配置工作流，但需要：
- 配置数据源 API（如聚宽、Tushare 等）
- 或我用脚本定时推送

### 方案 C：手动更新
直接在网页点击「编辑」按钮修改数据，或：
1. 编辑 portfolio-data.json
2. 修改相应字段（marketValue、price、dailyPnL 等）
3. Commit 后网页自动刷新

## ❓ 常见问题

**Q: 我没有 GitHub 账号怎么办？**
A: 可以先用 **飞书多维表格** 替代，我直接在多维表格更新数据。

**Q: 数据安全吗？**
A: GitHub 仓库可设为 Private，Pages 仍可访问。敏感信息（如资金账号）不会存储。

**Q: 能同步券商账户吗？**
A: 券商通常不开放 API。可每日截图发我，或手动维护数据。

**Q: 手机访问体验如何？**
A: 已做移动端适配，支持触屏操作和编辑。
